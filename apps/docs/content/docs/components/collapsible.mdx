## Installation

A component that allows content to be expanded or collapsed with smooth animations.

## Installation

The Collapsible component is built on top of `@rn-primitives/collapsible`. No additional installation is required as it comes pre-installed.

## Anatomy

```tsx
import {
  Collapsible,
  CollapsibleTrigger,
  CollapsibleContent,
} from '@/components/ui/collapsible'
```

## Usage

### Basic Collapsible

```tsx
import { useState } from 'react';
import { Text, View } from 'react-native';
import { Button } from '@/components/ui/button';
import { ChevronDown, ChevronUp } from 'lucide-react-native';
import {
  Collapsible,
  CollapsibleTrigger,
  CollapsibleContent,
} from '@/components/ui/collapsible';

function CollapsibleDemo() {
  const [isOpen, setIsOpen] = useState(false);

  return (
    <Collapsible open={isOpen} onOpenChange={setIsOpen}>
      <CollapsibleTrigger asChild>
        <Button variant="outline" className="w-full justify-between">
          <Text>Toggle Content</Text>
          {isOpen ? (
            <ChevronUp size={16} className="text-foreground" />
          ) : (
            <ChevronDown size={16} className="text-foreground" />
          )}
        </Button>
      </CollapsibleTrigger>
      <CollapsibleContent>
        <View className="space-y-2 px-4 py-3">
          <Text className="text-sm text-foreground">
            This content can be expanded and collapsed.
          </Text>
        </View>
      </CollapsibleContent>
    </Collapsible>
  );
}
```

## Props

### Collapsible

The root component that manages the collapsible state.

| Prop | Type | Description |
|------|------|-------------|
| `open` | `boolean` | The controlled open state of the collapsible |
| `defaultOpen` | `boolean` | The default open state when initially rendered |
| `onOpenChange` | `(open: boolean) => void` | Event handler called when the open state changes |
| `disabled` | `boolean` | When true, prevents the user from interacting with the collapsible |

### CollapsibleTrigger

The button that toggles the collapsible.

| Prop | Type | Description |
|------|------|-------------|
| `asChild` | `boolean` | Change the component to the HTML tag or custom component of the only child |
| `disabled` | `boolean` | When true, prevents the user from interacting with the trigger |

### CollapsibleContent

The component that contains the collapsible content.

| Prop | Type | Description |
|------|------|-------------|
| `forceMount` | `boolean` | Used to force mounting when more control is needed |
| `className` | `string` | Optional className for styling |

## Accessibility

The Collapsible component adheres to the WAI-ARIA design pattern for [Disclosure (Show/Hide)](https://www.w3.org/WAI/ARIA/apg/patterns/disclosure/).

- The trigger has role `button`
- The trigger has `aria-expanded` set automatically
- The trigger has `aria-controls` set automatically
- The content has role `region`
- The content has `aria-labelledby` set automatically
