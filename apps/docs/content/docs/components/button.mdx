## Usage

```tsx
import { Button } from '@/components/ui/button';
import { Text } from '@/components/ui/text';

<Button variant="default" size="default" onPress={() => console.log('Pressed')}>
  <Text>Click me</Text>
</Button>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| variant | `'default' \| 'secondary' \| 'destructive' \| 'outline' \| 'ghost' \| 'link'` | `'default'` | The visual style variant |
| size | `'default' \| 'sm' \| 'lg' \| 'icon'` | `'default'` | The size of the button |
| disabled | `boolean` | `false` | Whether the button is disabled |
| onPress | `() => void` | - | Handler called when button is pressed |
| className | `string` | - | Additional styles |

## Styling Tips

### Custom Variants

You can create custom button variants by using the className prop:

```tsx
// Success variant
<Button
  variant="default"
  className="bg-green-600 active:bg-green-700"
>
  <Text>Success</Text>
</Button>

// Warning variant
<Button
  variant="default"
  className="bg-yellow-600 active:bg-yellow-700"
>
  <Text>Warning</Text>
</Button>
```

### Rounded Buttons

```tsx
<Button variant="default" className="rounded-full">
  <Text>Rounded Full</Text>
</Button>

<Button variant="outline" className="rounded-lg">
  <Text>Rounded Large</Text>
</Button>
```

## Accessibility

- Buttons have proper touch targets (minimum 44x44 points)
- Disabled states are clearly indicated
- Focus states are visible for keyboard navigation
- Press states provide visual feedback
- Text contrast meets WCAG guidelines

## Best Practices

1. **Clear Labels**: Use descriptive text that explains the action
2. **Icon Usage**: Include icons to enhance meaning, not replace text
3. **Consistent Styling**: Use variants consistently across your app
4. **Loading States**: Show loading indicators for async actions
5. **Disabled Logic**: Disable buttons during loading or invalid states
6. **Touch Targets**: Ensure buttons are large enough for comfortable tapping
