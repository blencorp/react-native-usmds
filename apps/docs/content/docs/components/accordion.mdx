## Usage

```tsx
import {
  Accordion,
  AccordionContent,
  AccordionItem,
  AccordionTrigger,
} from '@/components/ui/accordion';
import { Text } from '@/components/ui/text';

<Accordion type="single" collapsible>
  <AccordionItem value="item-1">
    <AccordionTrigger>
      <Text>Is it accessible?</Text>
    </AccordionTrigger>
    <AccordionContent>
      <Text>Yes. It adheres to the WAI-ARIA design pattern.</Text>
    </AccordionContent>
  </AccordionItem>
</Accordion>
```

## Props

### Accordion

Extends `@rn-primitives/accordion` Root props.

| Prop | Type | Description |
|------|------|-------------|
| type | `'single' \| 'multiple'` | Determines whether one or multiple items can be opened at the same time. |
| collapsible | `boolean` | When type is "single", allows closing the open item. |
| value | `string \| string[]` | Controlled value of the item(s) to expand. |
| onValueChange | `(value: string \| string[]) => void` | Event handler called when value changes. |
| disabled | `boolean` | When true, prevents user interaction. |

### AccordionItem

| Prop | Type | Description |
|------|------|-------------|
| value | `string` | **Required.** A unique value for the item. |
| disabled | `boolean` | When true, prevents the item from being activated. |

### AccordionTrigger

| Prop | Type | Description |
|------|------|-------------|
| children | `React.ReactNode` | The content of the trigger. |
| className | `string` | Additional classes for styling. |

### AccordionContent

| Prop | Type | Description |
|------|------|-------------|
| children | `React.ReactNode` | The content to display when expanded. |
| className | `string` | Additional classes for styling. |
